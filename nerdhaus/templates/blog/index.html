collapsible top nav

7/30/2013

so as the site grows and develops, i'm implementing more and varied patterns for both the ui and the back end. today, i did a little research and banged out a responsive nav menu for the header up there.

(if you aren't sure what i'm talking about, drag the corner of your browser window around to resize the window and watch how things change.)

i made use of a straight forward technique that principally does two things:

  1. sets a class of active on the <ul> when the menu anchor is clicked.
  2. based on the presence of the active class, styles the max-height of the unordered list that contains the navigation anchors.

my approach uses jQuery for the application of the class and css transitions to make the whole effect smoother and more pallatable.

here's my markup, first: 

        <nav class="mobile">
            <ul id="#nav">
                <li><a href="http://nerdha.us"> Home </a></li>
                <li><a href="http://nerdha.us/blog"> Blog </a></li>
                <li><a href="http://nerdha.us/blag"> Blag </a></li>
                <li><a href="http://nerdha.us/shows"> Shows </a></li>
            </ul>
            <a class="nav-toggle" href="#nav"> Menu </a>
        </nav>

here's the javascript: 

        $(document).ready(function () {
            $('a.nav-toggle').click(function () {
                $('nav.mobile ul').toggleClass('active'); 
            });
        });

and the styles, processed with less: 

        nav.mobile {
            background: rgba(0, 0, 0, 0.9);
            overflow: hidden;
            
            a {
                border-bottom: 1px solid #000;
                color: @bg1;
                display: block;
                text-decoration: none;
                padding: 5px 0 5px 2em;
            }

            ul { 
                transition: max-height 0.4s ease-out 0s;
                visibility: hidden;
                max-height: 0;
                padding: 0; 
            }

            a.nav-toggle {
                float: right;
                padding-right: 2em;
            }
        }

        nav.mobile ul.active {
            max-height: 143px;
            visibility: visible;
        }

i've snipped out the rules that don't pertain to affecting the collapsible nav, so if you you want to get the exact same look and feel, i suggest you look at my source :]

in a nutshell, the javascript binds a click handler on the 'Menu' anchor that toggles a class on and off the unordered list that comprises the actual menu for the site. when it's active, the list smoothly expands to the max-height thanks to that transition declaration.

on conversation

7/16/2013

conversation is one of the most valuable resources that we engage on a daily basis. the act itself is valuable, regardless of the outcome. debate, discussion, anything that transcends the simple exchange of pleasantries and information (hi how are you i'm fine what can i get a triple yeah you want two please six fifty thanks).

even when you end up with a room (tangible or digital) full of pissed off people. there's a fine line, sure, because something must actually have been said for that emotion to be worth a damn. but by that same token: if somebody's pissed, it's more often for a reason. something was brought up that people think is worth getting pissed about. some thing worth a damn.

this martin-zimmerman business matters. it makes us talk. when we talk, we shape each others' opinions. we hash shit out.

Crash is a good movie. solid performances by talented people, a complicated but engaging yarn, and a meaningful shot at a potent, contemporary issue. but what really made that movie magical to me was it's bordering-on-realistic (for two hours worth of movie, anyway) examination of the simple causality that rules our lives.

a word there, the flick of a pen, a simple conversation. or a hard one. somebody's pissed, somebody goes home, somebody makes a phone call and your life is different. you can make it better, or you can make it worse.

people don't need to die. and when they do, it's worth making it a point not to ignore it, to blow it off. talk about it. at least talk about it.

this is your brain, organized

7/7/2013

i don't know about you, but i am seriously scatterbrained.

to be fair, i'm old school add, diagnosed in the early nineties (second grade, maybe?), and not just a spaz.

regardless, the way my mind works has forced me to come up with ways of keeping my shit straight, and one of the most powerful has been to write everything out. principally, for me, this means writing todo lists and keeping a wiki.

a simple todo list (i skip fancy tools here; an excel spreadsheet on google drive is way fancy enough) with columns for status, notes, and links anchors me to a set of things to pursue for a day. it also provides a nice way to dump things out of my mental space so that i can revisit them later. forgetfulness is an artform that i've spent many years refining, so anything that helps save brain cycles by remembering things for me is a huge boon.

now the wiki is my notebook, full of notes. i use it take document, retain, and reference so many things that i learn, mainly programming and business stuff. i take notes on things constantly. read a good article on css best practices? notes. pick up and start learning a new library? takin' notes. learn about a new pattern for solving a problem? note time.

the more i do this, the more i fall in love with it. it helps me track my learning arc through doing projects and picking up more of my craft.

as an added bonus, i keep it public so my pain can be your gain. this also encourages me to make it nice, real nice, so people will find it easy to digest and i'll never be bummed out using it.

right now i use moinmoin. i add to it constantly, mapping out what i know and use as i learn.

stupid simple flask app

7/6/2013

consider flask, a delightful web application framework, commonly referred to as a micro framework.

firing up a running web app in flask is cake. import the Flask class, instantiate an application object from it, define a route and view function, and call run() on your app object.


        from flask import Flask, render_template

        app = Flask(__name__)

        @app.route('/')
        def index():
            return render_template('index.html')

        if __name__ == '__main__':
            app.run()

easy peasy. this will serve the template called index.html at the root ('/') of the domain where the app is hanging out.

fab fun times

so as i continue on my magical journey to production quality flask, i discover my deep desire for a not dumb deployment solution. hence, fabric.

working with fabric, after some intial hiccups, is actually pretty delightful. it's a library created to facilitate interaction with remote systems via ssh. it picks up your .ssh/config and uses it right out of the box, and comes equipped with a simple vocabulary for moving around the filesystem and working with files.

i intend to use this lib quite a bit, so more in depth accounts of my experience doing so will follow.
